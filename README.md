# Petroleum-Data-Analysis-Dashboard
## Project Overview

This project presents a comprehensive data analysis of Saudi Arabia's petroleum sector, focusing on production trends, consumption patterns, export volumes, and reserves analysis. Leveraging Power BI, SQL Server, and advanced data modeling techniques, the dashboard delivers actionable insights to support strategic decision-making in the energy sector.
## Objectives
Clean and prepare petroleum datasets for analysis.

Build reusable SQL queries and views for reporting.

Create a structured data model for dashboarding.

Analyze trends in production, exports, refining activity, and demand.

Present findings through an interactive Power BI dashboard.

## Tools Used
**SQL Server** — data cleaning, transformation, joins, views, and analysis.

**Power BI** — data modeling, DAX measures, KPI tracking, and interactive dashboards.

**Excel / CSV**— source data format.

## Dataset Scope
The dataset includes petroleum-related indicators such as:

▸Crude oil production

▸Exports

▸Refinery intake

▸Direct use

▸Product demand

▸Reserves

## Project Workflow

1. Data Preparation in SQL
   
The raw data was prepared in SQL Server by:

   ▸ Standardizing column names and data types
   
   ▸ Cleaning inconsistent values
   
   ▸ Transforming dates into analysis-ready format
   
   ▸ Combining multiple tables where needed
   
   ▸ Creating reusable views for downstream reporting

2. Data Modeling in Power BI
   
After preparing the data in SQL, the cleaned tables were loaded into Power BI to:

   ▸ Build relationships between tables
   
   ▸ Create calculated measures using DAX
   
   ▸ Design KPI cards and trend visuals
   
   ▸ Enable filtering by time and category

3. Dashboard Development
   
The dashboard was designed to highlight:

   ▸ Monthly and yearly production trends
   
   ▸ Export patterns over time
   
   ▸ Refinery intake and direct use distribution
   
   ▸ Demand trends by product
   
   ▸ Reserve-related indicators
